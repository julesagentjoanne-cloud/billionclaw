# startupClaw - Product Integration - Community Agent

## 1. Agent / capability
- Community Agent

## 2. User-facing job
- design and run a community/economy around the startup so users, advocates, and ecosystem participants gain value and contribute back

## 3. Primary UI surfaces
- report section: community and ecosystem potential
- agent card: Community Agent
- workflow step: design community model
- dashboard block: motivations, rituals, champions, feedback themes
- asset/output screen: community strategy, ritual calendar, champion program
- review screen: community health and ecosystem insights

## 4. User flow impact
- appears when trust, education, advocacy, feedback, or category-building are strategic levers
- gives the founder a visible path from product/community idea to an operating community system

## 5. Key UI states
- not started
- community fit assessed
- archetype selected
- rituals designed
- launch-ready
- active
- under review
- blocked / no clear value proposition

## 6. Visible outputs
- community archetype
- motivation map
- rituals calendar
- champion framework
- feedback routing design
- community KPI starter set

## 7. Connected data/signals
- user base and ICP
- engagement and advocacy signals
- support/product feedback themes
- content engine outputs
- ecosystem goals

## 8. Cross-agent connections
- takes content support from Content Engine Agent
- routes feedback into Product, Messaging, and Onboarding/Retention logic
- can amplify GTM and category authority efforts

## 9. MVP version
- one community strategy view with archetype, member motivations, first rituals, and starter KPIs

## 10. Later expansion
- community dashboard
- champion management
- event/program builder
- product-feedback routing views
