# startupClaw - Product Integration - Content Engine Agent

## 1. Agent / capability
- Content Engine Agent

## 2. User-facing job
- create recurring content packages that grow audience, leads, and category authority on a schedule

## 3. Primary UI surfaces
- report section: content engine opportunity
- agent card: Content Engine Agent
- workflow step: schedule content production
- dashboard block: content pipeline and rules
- asset/output screen: posts, lead magnets, carousel packs, image prompts
- review screen: content performance and rule updates

## 4. User flow impact
- appears when content can be a major growth, trust, or ecosystem lever
- moves the founder from one-off post generation to a repeatable content engine inside the product

## 5. Key UI states
- not started
- sources connected
- hook research ready
- draft package ready
- needs review
- scheduled
- deployed
- learning rules updated
- blocked / missing voice or publishing access

## 6. Visible outputs
- hook shortlist
- post draft
- lead magnet draft/page
- carousel script
- image prompts
- schedule state
- QA/rule report

## 7. Connected data/signals
- founder voice sources
- content goals
- tracked influencer/competitor sources
- asset library
- content performance metrics
- manual edits and rules

## 8. Cross-agent connections
- feeds Community Agent and GTM Execution Agent
- uses Creative Agent for asset direction
- can be reviewed by Kill-Scale Agent for content theme scaling decisions

## 9. MVP version
- content package screen with hook research, one draft, one lead magnet, and approval/schedule state

## 10. Later expansion
- always-on content dashboard
- rule memory editor
- publishing integrations
- multi-channel content ops board
