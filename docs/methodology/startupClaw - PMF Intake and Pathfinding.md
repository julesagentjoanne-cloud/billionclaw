# startupClaw - PMF Intake and Pathfinding

## Purpose
Define the startupClaw intake mechanism that takes a founder's startup statement/value proposition, assesses the current startup state, and suggests multiple plausible paths toward product-market fit.

## Why this matters
This is one of startupClaw's most valuable mechanisms because it creates the front door into the whole venture-studio system.

It helps answer:
- what startup is this, really?
- what stage is it in?
- what is the likely bottleneck?
- what paths toward PMF are available?
- which path should be prioritized first?
- which expert/agent should work next?

## Core flow
### 1. Intake
The founder writes down the startup/value proposition using a simple input format.

Use:
- [[startupClaw - Business Statement Mode]]
- [[startupClaw - Input Template]]

### 2. Interpretation
startupClaw interprets:
- what is being built
- what problem is being solved
- who likely has the problem
- why someone might pay
- what alternatives exist
- what the likely audience/buyer/use-case set is

### 3. State assessment
startupClaw then estimates:
- likely stage
- likely bottleneck
- evidence vs assumption
- confidence level

Use:
- [[startupClaw - Decision Logic]]
- [[startupClaw - Product Data Model - Startup Workspace]]

### 4. Value proposition assessment
Assess:
- clarity
- urgency
- buyer strength
- trust burden
- commercial viability
- revenue proximity

### 5. PMF path generation
Generate multiple plausible paths to PMF.

Typical path types:
- sharper audience path
- stronger buyer path
- narrower use-case path
- better trust/proof path
- stronger offer path
- stronger channel path
- better message/category framing path

### 6. Path prioritization
Select one best path now based on:
- stage
- bottleneck
- confidence
- speed of learning
- commercial relevance
- execution realism

### 7. Agent routing
Map the selected path into the next best expert/agent and action module.

## Suggested founder-facing output
### 1. Startup interpretation
- what startupClaw believes the startup is

### 2. Current startup state
- likely stage
- likely bottleneck
- confidence level

### 3. Value proposition assessment
- what is strong
- what is weak
- what is still assumption

### 4. Suggested PMF paths
- Path A
- Path B
- Path C
with why each path could work

### 5. Recommended next move
- one best path now
- one best next agent
- one best next task

## Product role
In product terms, this mechanism should power:
- initial startup diagnosis
- startup workspace creation
- next-agent recommendation
- state update proposals as the startup evolves
