# startupClaw - Product Requirements Document

## Purpose
Define startupClaw clearly enough that a coding AI agent can build it into a real product and prepare it for launch.

## Product summary
startupClaw is a founder product that turns a simple business statement into:
- a startup intelligence report
- audience expansion ideas
- alternative buyer mapping
- demand-signal analysis
- founder-channel fit
- initial wedge vs expansion path
- why-now story
- founding insight
- category story
- distribution advantage
- trust architecture needs
- stage and bottleneck diagnosis
- one best next move
- optional buyable execution agents that produce useful startup assets

## Core promise
Turn one startup idea statement into a report and execution path that helps a founder actually move.

## Target user
### Primary user
- early-stage founder
- indie hacker
- solo founder
- small startup team

### Secondary user
- advisor / operator helping startups
- incubator/studio user
- GTM consultant

## Primary use case
Founder enters a short business statement.
startupClaw interprets it, expands it, diagnoses it, and returns:
- a strong report
- one best next move
- recommended execution agents

## Inputs
### Default input mode
Use [[startupClaw - Business Statement Mode]]:
- we are building X for Y
- they currently use Z
- they should choose us because A
- they would pay because B
- optional evidence

### Optional deeper input mode
Use [[startupClaw - Input Template]] if more detail is needed.

## Core outputs
Use [[startupClaw - Full Report Template]]:
- startup interpretation
- existing demand evidence
- audience expansion
- alternative buyer mapping
- persona + urgency
- revenue proximity
- competitor / alternative map
- stage + bottleneck
- best next move
- agent execution options
- what startupClaw did

## Core product flow
1. founder enters business statement
2. system interprets idea
3. system expands audiences and buyers
4. system diagnoses stage and bottleneck
5. system generates report
6. system recommends one execution agent
7. founder can buy/select agent help
8. system generates execution asset(s)
9. system supports review loop

## Product decisions
### UX rule
Keep the surface simple:
- one idea statement in
- one report out
- one next move
- one recommended agent

### System rule
Backend can be orchestrated and modular.
Frontend should feel simple and fast.

### Trust rule
Every report should include:
- what startupClaw inferred
- what signals it used
- what still needs validation

## Success criteria
### User-level
- founder gets to useful insight quickly
- founder feels report is sharper than their own first framing
- founder understands what to do next
- founder sees value in optional execution agents

### Product-level
- high completion rate from input to report
- strong report shareability
- users selecting at least one execution agent
- repeat usage through review loop

## Launch requirement
Product should be launchable in a way that supports Product Hunt.
That means:
- clear homepage
- report demo example
- one click into product flow
- one clear product story
