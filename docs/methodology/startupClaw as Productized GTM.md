# startupClaw as Productized GTM

## Purpose
Make explicit that startupClaw is the productized front-end of the broader GTM advisory system.

## Simple overview
startupClaw is not a separate idea from GTM advisory.
It is the productized version of the same core logic.

In simple terms:
- **startupClaw** = self-serve / guided product layer
- **GTM advisory** = higher-touch expert/service layer
- **shared core** = startup diagnosis + GTM reasoning engine

## Core idea
Build one underlying system that can be used in two ways:

### 1. By founders directly
Through startupClaw:
- describe what they are building
- describe the problem they solve
- get audience/pain suggestions
- get startup-stage guidance
- get next-step GTM guidance

### 2. By Jules in GTM advisory
Through advisory work:
- assess startups more deeply
- diagnose bottlenecks
- generate better roadmaps
- speed up research and prioritization
- create more repeatable client delivery

## Shared core system
The same core logic should power both layers:
- founder input capture
- pain-point research
- ICP research
- alternative / competitor research
- startup stage detection
- bottleneck diagnosis
- scorecard logic
- next-step recommendation logic
- GTM setup logic

## Why this is the right architecture
If startupClaw and GTM advisory become separate systems, the work duplicates.

Better structure:
- one intelligence layer
- two delivery modes

That means startupClaw becomes:
- a productized interface for founders

And GTM advisory becomes:
- an expert-led use of the same system with more depth and context

## Relationship to other notes
### startupClaw
Covers the founder-facing product flow.

### GTM Advisory Operating System
Covers the broader advisory work and service logic.

### Shared research and diagnostic notes
Provide the reusable intelligence layer beneath both.

## Practical implication
When designing notes, frameworks, and tools, ask:
1. Is this a shared core capability?
2. Is this startupClaw-specific UX/product logic?
3. Is this GTM advisory-specific delivery logic?

## Rule
Default to building **shared core logic first**.
Then decide how it appears in:
- startupClaw
- GTM advisory
