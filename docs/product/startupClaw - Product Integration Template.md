# startupClaw - Product Integration Template

## Purpose
Template for translating a startupClaw agent from system logic into visible product behavior.

## Core rule
Every meaningful agent should eventually have both:
- stored logic/instructions
- product expression in the UI

## Template
### 1. Agent / capability
- name of agent or system logic

### 2. User-facing job
- what the founder/user thinks this does

### 3. Primary UI surfaces
- report section
- agent card
- workflow step
- dashboard block
- asset/output screen
- review/monitoring screen

### 4. User flow impact
- where this appears in the journey
- what it changes in input -> analysis -> execution -> review

### 5. Key UI states
- not started
- running
- draft ready
- needs review
- completed
- blocked / low confidence

### 6. Visible outputs
- what the user can read, click, edit, export, or act on

### 7. Connected data/signals
- what data powers this UI surface

### 8. Cross-agent connections
- what other agents it feeds or depends on

### 9. MVP version
- smallest useful UI expression

### 10. Later expansion
- richer dashboards, automation, collaboration, review loops
