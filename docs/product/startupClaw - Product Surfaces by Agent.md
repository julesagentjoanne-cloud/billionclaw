# startupClaw - Product Surfaces by Agent

## Purpose
Group startupClaw agents into shared UI/product surfaces so the venture-builder product stays understandable.

## Core product surfaces
### 1. Report
Shows:
- diagnosis
- recommended expert agent
- why this agent now
- visible next actions

### 2. Agent catalog
Shows:
- expert role
- domain owned
- skill stack
- what it produces
- when to use it

### 3. Execution workspace
Shows:
- active task state
- inputs available
- assets being generated
- what the AI is doing
- what the user still needs to provide

### 4. Output/asset screens
Shows:
- generated work product
- how to use it
- status of the task
- edit/export next actions

### 5. Review/dashboard layer
Shows:
- performance state
- blocked states
- next recommendations
- cross-agent follow-up

### 6. Decision / handoff layer
Shows:
- the recommended next agent or module
- why that handoff is being made
- whether the next step is fully executable now, partially executable now, or blocked
- what founder approval or missing input is required before the handoff runs

This matters because startupClaw should not stop at diagnosis. The product should make the transition from insight to next action visible and editable.

## Surface expansion note
- [[startupClaw - Research Agent]] should surface as an evidence workspace, not just a static brief.
- [[startupClaw - Validation Agent]] should surface as an assumption board with test status, because its value is in visible learning progression.

## Surface-specific note
- [[startupClaw - Messaging Agent]] should not stay buried inside a static report.
- In product, its value is clearest when founders can compare message variants, inspect proof strength, and approve which version moves into execution.

## Product rule
The UI should always make it simple for the founder to understand:
- what startupClaw is doing
- what is already done
- what is blocked
- what input is expected from the founder
