# startupClaw - Product UI Mapping Checklist

## Purpose
Quick checklist to make sure new startupClaw logic becomes visible product behavior.

## Checklist
When adding a new agent/capability, check:
- Does it need a new report section?
- Does it need a new or updated agent card?
- Does it create a new workflow step?
- Does it produce a visible asset/output?
- Does it need a dashboard/review state?
- Does it need user-editable inputs or filters?
- Does it need approval/review states?
- Does it connect to another agent visibly?
- Does it need a lightweight MVP UI now?
- What would make the value obvious to a founder in the product?

## Product rule
Do not let important logic remain backend-only if it creates real founder value.
