# startupClaw - Input Template

## Purpose
Provide the founder-facing starting input for startupClaw in a way that creates a fast aha-moment, supports self-serve progress, and feeds the deeper startup diagnosis / GTM system when needed.

## Simple overview
This template is designed to capture the startup **as-is**.

It should help founders:
- describe what they are building
- describe what problem they think they solve
- share what they already believe
- leave blanks where they are unsure
- get research-backed suggestions to move forward

The goal is not to force perfect answers.
The goal is to create enough signal for startupClaw to:
- research the market
- suggest likely audiences, pains, and alternatives
- identify likely stage and bottleneck
- recommend the next best GTM move

## Belief statements section
Use these if helpful. Leave blanks empty if you do not know yet.

### 1. Audience / pain belief
We believe **[target audience]** will care because they need to get **[job to be done]** done / solve **[pain]**.

### 2. Willingness-to-pay belief
We believe they will pay **[price / range]** because this helps them achieve **[outcome]** or avoid **[cost / pain]**.

### 3. Alternative belief
Instead of **[competitor 1 / competitor 2 / workaround]**, customers may choose us because **[difference / advantage]**.

### 4. Channel belief
We believe the best early GTM path is **[channel]** because **[reason]**.

### 5. Proof belief
We think this might be true because **[evidence / signal]**.

## How startupClaw should use this input
After receiving the founder input, startupClaw should:
1. identify what is clear
2. identify what is missing
3. identify what is assumption vs evidence
4. use [[startupClaw - Research Inputs]] to research likely audiences, pains, alternatives, and willingness-to-pay clues
5. return structured options instead of a research dump
6. guide the founder toward the next best question or GTM step

## Product rule
This template should feel like a **starting prompt**, not a heavy form.

That means:
- only ask for the minimum useful input first
- allow blanks
- let research fill options where possible
- help the founder move forward from the current state
- only escalate into deeper intake when necessary

## Advisory rule
This same template should also be usable in GTM advisory.

That means:
- it should capture the startup as-is
- it should surface the biggest unknowns quickly
- it should be easy to use in live advisory sessions
- it should feed naturally into [[Startup Intake Questionnaire]] and [[Startup Assessment Scorecard]] when deeper work is needed
