# startupClaw - Output Template

## Purpose
Provide the standardized output format startupClaw should return after processing founder input and external research.

## Simple overview
The output should help a startup move forward from its **current reality**.

It should:
- create a fast aha-moment
- clarify what seems most likely
- separate evidence from assumption
- show where the startup likely is in the journey
- recommend one practical next move

The output should not feel like a vague report.
It should feel like a founder-friendly decision aid.

## Product rule
Default to the **simple snapshot first**.
Only expand into the detailed layer if:
- extra context is useful
- the founder asks for more
- the startup is complex enough to warrant it

## Advisory rule
This output format should also work in GTM advisory mode.
That means it should:
- be readable in a live session
- be good enough for a founder to act on immediately
- also feed naturally into deeper diagnosis, scoring, and roadmap work
