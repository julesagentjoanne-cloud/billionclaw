# startupClaw - Action Modules - Messaging

## Purpose
Define messaging-oriented action modules startupClaw can execute when positioning and message clarity are limiting progress.

## Simple overview
Use these modules when the startup needs to make the value clearer, sharper, and more compelling.

## Action modules
### 1. Positioning angle generator
- generate 3 positioning angles
- compare problem-first vs outcome-first frames

### 2. Value proposition rewrite
- rewrite in plain language
- reduce jargon and vague claims

### 3. Landing page angle draft
- generate headline / subhead / CTA variants
- align the page to a high-intent pain angle

### 4. Objection handling draft
- identify likely objections
- draft clearer trust / proof / differentiation language

## Best use cases
- positioning bottleneck
- low response to messaging
- weak landing-page conversion
- unclear differentiation

## Module selection rules
Prefer this module when:
- the audience and use case are clear enough to craft targeted messaging
- the startup's core issue is comprehension, resonance, or differentiation
- improving message clarity is likely to unlock better conversion or stronger GTM tests

Do not use this module first when:
- the startup still lacks a credible problem or ICP hypothesis
- the main unknown is willingness to pay rather than explanation quality
- weak performance is mostly caused by channel mismatch or offer weakness

## Output rule
Each messaging output should specify:
- the audience/use case it is written for
- the core problem or desired outcome being emphasized
- the proof or trust element needed
- where the message should be used first
- whether this should map into product UI as a report section, comparison view, or editable asset
- the confidence level and the next test needed if evidence is still weak

## Product UI mapping
- should map into product UI: yes
- founder should see side-by-side message variants, the variable being tested, confidence level, and a simple approval state for what ships next
- this should be visible in both the report and an editable messaging asset view
