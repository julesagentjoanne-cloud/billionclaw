# startupClaw - Asset Generators

## Purpose
Define the execution assets startupClaw should be able to generate after diagnosis and recommendation.

## Simple overview
To behave like an operator, startupClaw should not stop at recommendations.
It should generate the assets the founder needs to move.

## Core asset types
### 1. Validation assets
- interview script
- survey questions
- pain-validation prompt set
- willingness-to-pay test script

### 2. Messaging assets
- value proposition variants
- headline variants
- landing page angle drafts
- objection-handling bullets

### 3. Offer assets
- package/tier draft
- pricing test plan
- smallest useful offer definition

### 4. Onboarding / retention assets
- onboarding email sequence draft
- getting-started checklist
- first-week journey map
- retention interview script

### 5. Channel assets
- channel test plan
- outreach copy draft
- content angle matrix
- message-channel fit test sheet

### 6. Founder / operations assets
- founder bottleneck audit
- delegation matrix
- SOP starter outline
- recurring-task checklist

## Generation rule
Asset generation should be tied to:
- stage
- bottleneck
- current recommendation
- what startupClaw already inferred

## Output rule
When startupClaw creates assets, it should say:
- what it generated
- why this asset is the next useful asset
- how the founder should use it
- what result to watch for
