# startupClaw - Coding Agent Brief

## Goal
Build startupClaw as an MVP product that can be deployed and prepared for Product Hunt.

## What startupClaw does
startupClaw takes a founder's business statement and generates:
- a startup intelligence report
- audience expansion
- alternative buyer mapping
- demand evidence interpretation
- stage and bottleneck diagnosis
- one best next move
- optional execution-agent outputs

## What to build first
### MVP
- landing page
- business statement form
- report page
- agent recommendation section
- asset generator output section
- one demo/example case

## Required product behaviors
- input should be simple
- report should be readable and impressive
- one best next move should stand out
- one recommended execution agent should stand out
- generated asset should feel immediately useful
- report should include a trust/transparency section (“what startupClaw did”)

## Required pages
- landing page
- input page
- report page
- asset page

## Required first asset generators
Build at least these:
- interview script generator
- landing page draft generator
- pricing test plan generator

## UI standard
- clean
- modern
- founder-facing
- screenshot-friendly
- not overly technical

## Technical preference
- Next.js
- TypeScript
- Tailwind
- Vercel-ready
- simple persistence only if needed

## Product rule
Bias toward:
- simplicity
- usefulness
- speed
- demo quality

Do not bias toward:
- overbuilding
- complex auth/collaboration
- enterprise complexity

## Definition of done
Done means:
- deployed app works end to end
- the main product story is obvious
- a founder can go from idea statement to report to useful generated asset
- Product Hunt launch assets are prepared
