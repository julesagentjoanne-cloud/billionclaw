# startupClaw - Product Integration - GTM Execution Agent

## 1. Agent / capability
- GTM Execution Agent

## 2. User-facing job
- turn diagnosis into an executable go-to-market move with concrete assets and a clear test plan

## 3. Primary UI surfaces
- report section: GTM next move
- agent card: GTM Execution Agent
- workflow step: launch experiment
- asset/output screen: channel test plan, outreach pack, funnel experiment
- review screen: GTM experiment results and next actions

## 4. User flow impact
- appears after diagnosis when acquisition/GTM is the main bottleneck
- moves the founder from analysis into execution without leaving the product

## 5. Key UI states
- not started
- hypothesis selected
- assets generating
- draft ready
- launched
- results under review
- blocked / missing data

## 6. Visible outputs
- selected channel
- audience/message pairing
- launch checklist
- outreach/campaign drafts
- experiment success signal
- next recommended step

## 7. Connected data/signals
- ICP and buyer mapping
- messaging recommendations
- offer context
- channel assumptions
- experiment performance data

## 8. Cross-agent connections
- depends on Messaging Agent and Offer Agent inputs
- feeds Kill-Scale Agent once tests are live
- can request Creative Agent assets for campaigns

## 9. MVP version
- report CTA into a GTM Execution Agent detail view with one experiment plan and downloadable assets

## 10. Later expansion
- multi-channel experiment board
- live campaign tracking
- auto-routing into Kill-Scale review
