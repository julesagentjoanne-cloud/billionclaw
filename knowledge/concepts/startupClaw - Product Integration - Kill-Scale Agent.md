# startupClaw - Product Integration - Kill-Scale Agent

## 1. Agent / capability
- Kill-Scale Agent

## 2. User-facing job
- review live campaign performance and tell the founder what to kill, fix, watch, scale, or exploit further

## 3. Primary UI surfaces
- report section: growth review readiness
- agent card: Kill-Scale Agent
- workflow step: review live experiments
- dashboard block: campaign decision states
- review/monitoring screen: campaign triage board

## 4. User flow impact
- appears after GTM experiments or campaigns exist
- turns performance data into clear action instead of raw reporting

## 5. Key UI states
- no live data
- data syncing
- triage in progress
- decision board ready
- needs review
- actions accepted
- blocked / insufficient confidence

## 6. Visible outputs
- campaign state labels
- confidence notes
- fatigue flags
- budget reallocation suggestions
- next experiment backlog

## 7. Connected data/signals
- campaign metrics
- audience metrics
- funnel conversion
- revenue outcomes
- prior creative winners

## 8. Cross-agent connections
- takes GTM Execution Agent outputs as inputs once live
- routes fixes to Creative Agent, Messaging Agent, or Offer Agent depending on diagnosis
- informs dashboard-level growth decisions

## 9. MVP version
- triage table with kill/fix/watch/scale states and a short reasoning column

## 10. Later expansion
- full campaign review dashboard
- trend charts
- automated alerts and recommended next asset requests
