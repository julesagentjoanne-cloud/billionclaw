# startupClaw - Action Modules - Growth Execution

## Purpose
Define growth-execution action modules startupClaw can execute when the startup needs a practical GTM move.

## Simple overview
Use these modules when the startup needs to turn diagnosis into a concrete growth experiment.

## Action modules
### 1. Channel test plan
- generate first-channel experiment plan
- define hypothesis, test, and success signal

### 2. Outreach / campaign draft
- generate outreach draft or simple campaign angles
- align messaging to the likely pain/buyer

### 3. Content angle matrix
- generate content or distribution angles by pain / audience / channel

### 4. Funnel experiment plan
- generate a focused improvement plan for a weak funnel step

## Best use cases
- repeatable acquisition
- acquisition bottleneck
- channel selection questions
- need for a first GTM experiment

## Module selection rules
Prefer this module when:
- the startup already has a plausible ICP, offer, and message hypothesis
- the main unknown is channel execution or audience-message pairing
- a live market test can generate better signal faster than more analysis

Do not use this module first when:
- the problem or audience is still too unclear for a fair GTM test
- the message is too weak or generic to support execution
- the startup needs validation before traffic or outreach scale

## Output rule
Each growth-execution output should specify:
- the single experiment being prioritized
- the audience
- the message angle
- the channel
- the success signal
- the decision rule for what to do next if the test wins, loses, or is inconclusive

## Product UI mapping note
This module should map into product UI as:
- a launch-experiment workflow step
- an execution workspace for active GTM tasks
- a review state that hands results into [[startupClaw - Kill-Scale Agent]]
