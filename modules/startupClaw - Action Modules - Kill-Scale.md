# startupClaw - Action Modules - Kill-Scale

## Purpose
Define the campaign triage action modules startupClaw can execute when the startup needs to decide what to kill, fix, watch, scale, or exploit further.

## Simple overview
Use these modules when the startup already has campaigns, ads, or GTM experiments running and needs sharper decisions.

This module should convert performance data into action.
Not just reporting.

## Core decision states
- **Kill** -> weak signal, poor economics, or low strategic value
- **Fix** -> core hypothesis may still be right, but execution is weak
- **Watch** -> still too early or noisy to make a confident call
- **Scale** -> strong enough to increase spend or exposure
- **Exploit further** -> proven winner; branch into more variants, audiences, or channels

## Action modules
### 1. Campaign decision scorecard
- score campaigns on efficiency, quality, economics, and strategic fit
- classify each campaign into kill / fix / watch / scale / exploit further

### 2. Budget reallocation plan
- shift budget away from weak tests
- increase investment in high-confidence winners
- protect small exploratory tests where signal is still forming

### 3. Creative fatigue analysis
- identify ad fatigue, declining hooks, or overused formats
- distinguish creative decay from audience or landing-page issues

### 4. Audience fatigue analysis
- identify segment saturation, narrow targeting exhaustion, or rising acquisition costs
- suggest refreshes, exclusions, or adjacent audiences

### 5. Failure diagnosis pack
- explain whether the failure is likely caused by creative, audience, offer, landing page, funnel handoff, or bad fit
- suggest the smallest useful next fix before killing a test

### 6. Winner exploitation plan
- define what to clone from current winners
- expand across angles, audiences, formats, placements, or channels

## Inputs
- campaign-level performance data
- ad-level performance data
- audience-level performance data
- funnel and conversion data
- CRM / revenue outcomes when available
- benchmarks or target thresholds
- prior winning messages and creatives

## Outputs
- campaign decision table
- kill/fix/watch/scale recommendations
- reallocation suggestions
- next experiment backlog
- fatigue report
- confidence notes

## Best use cases
- many campaigns running but unclear what to do next
- rising costs with inconsistent performance
- unclear whether a campaign should be killed or fixed
- need to direct spend toward winners faster
