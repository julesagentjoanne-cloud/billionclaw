# startupClaw - Action Modules - Operator Systems

## Purpose
Define operator/system action modules startupClaw can execute when internal leverage, workflow, or founder dependency is the bottleneck.

## Simple overview
Use these modules when the startup is being slowed down more by internal execution constraints than by market demand.

## Action modules
### 1. Founder bottleneck audit
- identify what depends too heavily on the founder
- identify what only the founder should still own

### 2. Delegation matrix
- sort work into founder-only, delegatable, and systematizable

### 3. SOP starter pack
- generate simple SOP outlines/checklists for recurring work

### 4. Recurring task cleanup
- identify broken handoffs, repeated confusion, and manual drag

## Best use cases
- founder bottleneck
- operations bottleneck
- scaling chaos
- service/business model delivery drag
